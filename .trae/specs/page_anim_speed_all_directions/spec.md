# 全部翻页动画速度调节 - 产品需求文档

## 概述
- **Summary**: 实现所有翻页动画类型在长按翻页区域时显示速度调节弹窗的功能，并支持6个方位（左上、左中、左下、右上、右中、右下）的独立速度设置。
- **Purpose**: 解决当前只有仿真二翻页动画支持速度调节的问题，为所有翻页动画类型提供统一的速度调节功能，提升用户阅读体验。
- **Target Users**: 所有使用 Legado with MD3 应用进行阅读的用户。

## 目标
- 为所有翻页动画类型（覆盖、滑动、仿真、仿真二、滚动、渐变、无动画）添加速度调节功能
- 实现长按翻页区域时显示速度调节弹窗
- 支持6个方位（左上、左中、左下、右上、右中、右下）的独立速度设置
- 保持弹窗样式与现有项目UI风格一致
- 确保速度调节范围为10-500ms

## 非目标（范围外）
- 不修改现有的翻页动画核心算法
- 不添加新的翻页动画类型
- 不改变现有的UI布局结构

## 背景与上下文
- 项目中已实现了仿真二翻页动画的速度调节功能
- 其他翻页动画类型目前没有速度调节功能
- 现有的速度调节弹窗只支持单一速度设置，不支持按方位设置
- 项目使用9宫格布局（3x3）来划分屏幕区域，包括：
  - 左上 (tl)
  - 左中 (ml)
  - 左下 (bl)
  - 右上 (tr)
  - 右中 (mr)
  - 右下 (br)
  - 中上 (tc)
  - 中中 (mc)
  - 中下 (bc)

## 功能需求
- **FR-1**: 为所有翻页动画类型添加速度调节功能
- **FR-2**: 实现长按任意翻页区域时显示速度调节弹窗
- **FR-3**: 支持6个方位（左上、左中、左下、右上、右中、右下）的独立速度设置
- **FR-4**: 速度调节范围为10-500ms，步长为10ms
- **FR-5**: 保存用户的速度设置，并在应用重启后保持
- **FR-6**: 弹窗样式与现有项目UI风格保持一致

## 非功能需求
- **NFR-1**: 弹窗布局美观、易用，符合Material Design 3设计规范
- **NFR-2**: 速度调节响应及时，无明显延迟
- **NFR-3**: 不影响应用整体性能
- **NFR-4**: 代码实现符合项目现有代码风格

## 约束
- **Technical**: Android 平台，Kotlin 语言，使用现有项目框架
- **Dependencies**: 依赖现有项目的动画系统和UI组件

## 假设
- 用户已经熟悉现有的翻页操作和设置界面
- 项目已经有完善的设置存储机制
- 现有的弹窗组件可以用于实现速度调节功能

## 验收标准

### AC-1: 所有翻页动画类型支持速度调节
- **Given**: 用户选择任意翻页动画类型
- **When**: 用户长按翻页区域
- **Then**: 应显示速度调节弹窗
- **Verification**: `human-judgment`

### AC-2: 6个方位支持独立速度设置
- **Given**: 用户在不同方位长按
- **When**: 用户调整速度并保存
- **Then**: 不同方位应保持各自的速度设置
- **Verification**: `programmatic`

### AC-3: 速度调节范围正确
- **Given**: 用户打开速度调节弹窗
- **When**: 用户拖动滑块
- **Then**: 速度应在10-500ms范围内调节
- **Verification**: `human-judgment`

### AC-4: 速度设置持久化
- **Given**: 用户调整速度并关闭应用
- **When**: 用户重新打开应用
- **Then**: 速度设置应保持不变
- **Verification**: `programmatic`

### AC-5: 弹窗样式一致
- **Given**: 速度调节弹窗显示
- **When**: 用户观察弹窗样式
- **Then**: 弹窗应与现有项目的其他弹窗样式保持一致
- **Verification**: `human-judgment`
